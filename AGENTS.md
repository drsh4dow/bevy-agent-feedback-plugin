# Context

## General Rules

When implementing something always check what is the best modern way of
implementing this in the current version of bevy used on the repo.

Your knowledge is stale, so research before assuming something.

## Code Design

Prefer modularization through plugins to make deep modules.

## CI Gate

Run format/linting/tests after changes

## Communication rules

Be extremely concise. Sacrifice grammar for the sake of concision.
Less text, less code, is always better than more.
